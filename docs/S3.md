# Загрузка файлов в S3

Mivra умеет загружать картинки и документы напрямую в S3-совместимое облако. После загрузки в документ автоматически вставляется готовая markdown-ссылка с публичным URL — получатель `.md`-файла видит изображения из интернета, не надо передавать локальную папку `assets/`.

## Что поддерживается

**Провайдеры:** любые S3-совместимые. Тестировано на:

- **Yandex Object Storage** — `https://storage.yandexcloud.net`, регион `ru-central1`
- **TimeWeb Cloud Storage** — `https://s3.twcstorage.ru`, регион `ru-1`
- **AWS S3** — `https://s3.<region>.amazonaws.com` (поле endpoint можно оставить пустым только если у вас именно AWS — иначе обязательно)
- **Cloudflare R2, Selectel, Backblaze B2, MinIO, DigitalOcean Spaces** — должны работать «из коробки» с правильным endpoint

**Типы файлов:** картинки (`png/jpg/jpeg/gif/webp/svg/bmp/apng/avif/tiff/heic`) и документы (`pdf/zip/mp4/webm`). Остальные форматы отклоняются с сообщением «расширение не поддерживается».

**Лимиты:** до 100 MB на файл (хард-лимит). На файлах >10 MB Mivra спросит подтверждение перед загрузкой.

## Настройка

### Шаг 1 — Получить ключи у провайдера

Вам нужны 4 значения:

1. **Endpoint URL** — адрес S3-API провайдера.
2. **Region** — регион хранилища.
3. **Bucket** — имя bucket'а, в который грузим.
4. **Access Key ID** + **Secret Access Key** — пара ключей доступа.

#### Yandex Object Storage

1. Зайдите в [Yandex Cloud Console](https://console.cloud.yandex.ru/) → ваш каталог → **Object Storage**.
2. Создайте bucket (либо используйте существующий). Запомните имя.
3. В разделе **Сервисные аккаунты** создайте аккаунт с ролью `storage.editor`. Сгенерируйте **статический ключ** — это пара Access Key ID + Secret.
4. Endpoint: `https://storage.yandexcloud.net`. Регион: `ru-central1`.

#### TimeWeb Cloud Storage

1. В панели TimeWeb откройте **Облачное хранилище** → ваш bucket.
2. В сайдбаре справа найдите блок с реквизитами:
   - **S3 URL** — это endpoint (`https://s3.twcstorage.ru`)
   - **Название бакета** — это поле `Bucket` в Mivra
   - **S3 Access Key** + **S3 Secret Access Key** — пара ключей
   - **Регион** — `ru-1`
3. ⚠️ **Не используйте Swift-ключи** (Swift Access Key / Swift Secret) — это другой протокол, Mivra работает только с S3.

#### AWS S3

1. В консоли AWS создайте bucket в нужном регионе.
2. В IAM создайте пользователя с программным доступом и политикой `AmazonS3FullAccess` (или ограничьте конкретным bucket'ом).
3. Endpoint: `https://s3.<region>.amazonaws.com` (например, `https://s3.eu-central-1.amazonaws.com`). Регион: код региона (`eu-central-1`, `us-east-1` и т. п.).

### Шаг 2 — Включить публичный доступ к bucket

В markdown вставляется **публичная** ссылка. Если bucket приватный — у получателя файла откроется 403 на картинке.

- **Yandex:** настройки bucket → **Доступ** → выбрать «Публичный для чтения объектов».
- **TimeWeb:** настройки bucket → включить «Публичный доступ» / «Анонимное чтение».
- **AWS:** установить bucket policy с `s3:GetObject` для `Principal: *` либо включить ACL.

### Шаг 3 — Заполнить настройки в Mivra

1. Откройте Mivra → нажмите кнопку **«S3»** в правом верхнем углу (рядом с «Справка»).
2. Заполните поля:

| Поле в Mivra | Что вводить |
|---|---|
| **Endpoint URL** | URL S3-API провайдера |
| **Регион** | код региона |
| **Bucket** | имя bucket'а |
| **Access Key ID** | публичная часть ключа |
| **Access Key Secret** | секретная часть (хранится в Windows Credential Manager, не в файле) |
| **Public URL prefix** *(опционально)* | если у bucket'а есть отдельный публичный домен / CDN. Если пусто — Mivra соберёт URL автоматически из endpoint+bucket |
| **Path prefix** *(опционально)* | папка внутри bucket для файлов из Mivra, например `mivra/`. Чтобы не сваливать в корень |

3. Нажмите **«Тест соединения»**. Должно появиться зелёное «Соединение работает».
4. Нажмите **«Сохранить»**.

После сохранения кнопка **«S3»** в Toolbar загорится **зелёным** — значит конфиг рабочий и Mivra готов грузить.

### Что делать, если кнопка не зелёная

Зелёная подсветка появляется только когда:
- Все обязательные поля заполнены
- В этой же сессии диалога успешно прошёл «Тест соединения»
- Нажата «Сохранить»

Если кнопка серая после сохранения — снова откройте «S3», нажмите «Тест соединения», и потом «Сохранить». Mivra специально не зажигает зелёный без явного теста — чтобы не вводить в заблуждение, что данные точно рабочие.

## Как загружать файлы

После настройки S3 файлы можно вставлять **тремя способами**:

### 1. Перетаскивание из проводника (Drag & Drop)

Просто перетащите файл (или несколько файлов) из проводника Windows прямо в окно Mivra. Каждый файл загрузится по очереди, в редакторе появится строка `![имя](https://...)` для картинок или `[имя](https://...)` для документов.

### 2. Вставка из буфера обмена (Ctrl+V)

- Сделайте скриншот через **Win+Shift+S** (или любым другим способом — главное, чтобы картинка была в буфере).
- Кликните в редактор Mivra и нажмите **Ctrl+V**.
- Mivra перехватит вставку картинки, загрузит на S3 и вставит ссылку. Имя файла будет автогенерироваться (`clipboard-1714500000000.png`).

### Если S3 не настроен или не прошёл «Тест соединения»

В таком случае Mivra **не пытается** грузить в облако (никаких ошибок «не удалось подключиться»). Вместо этого все три триггера работают **локально**:

- **Drag & drop** → файл копируется в `{папка_документа}/assets/{имя}`, в документ вставляется относительная ссылка `![имя](assets/имя.png)`. Если в `assets/` уже есть файл с таким именем — Mivra добавит `(1)`, `(2)`, и т. д.
- **Ctrl+V** → картинка из буфера сохраняется в `{папка_документа}/assets/clipboard-{timestamp}.{ext}`.
- **Кнопка Ctrl+Shift+A** → открывается диалог выбора файла, относительная ссылка вставляется напрямую.

**Условие:** документ должен быть **сохранён** на диск (чтобы Mivra знал, в какую папку класть `assets/`). Если файл новый и ещё не сохранён — Mivra покажет тост «Сохраните документ, чтобы добавлять локальные файлы».

Чтобы перейти в облачный режим — настройте S3 и нажмите «Тест соединения». Кнопка «S3» в Toolbar загорится зелёным.

### 3. Кнопка «Вставить файл» (Ctrl+Shift+A)

- В Toolbar нажмите **«Вставить файл»** или горячую клавишу **Ctrl+Shift+A**.
- Откроется системный файловый диалог. Выберите файл.
- Mivra загрузит его и вставит ссылку.

## Что увидит пользователь

Во время загрузки появится **toast-уведомление** в правом нижнем углу:

- 🔄 «Загрузка `имя.png`...» — идёт upload
- ✅ «Готово» — после успеха (исчезает через 2 секунды)
- ❌ «Ошибка: ...» — если что-то пошло не так (остаётся, пока не закроете)

Markdown-ссылка вставляется **в текущую позицию курсора** (в source-режиме) или **в конец документа** (в visual-режиме).

## Безопасность

- **Secret Access Key** хранится в **Windows Credential Manager** (запись `Mivra` / `s3_secret_access_key`), не в файле `settings.json`. При переустановке Windows / смене учётки придётся ввести заново — endpoint/region/bucket остаются.
- В файле `settings.json` хранится только **Access Key ID** (публичная часть, его утечка не критична).
- Кнопка **«Очистить»** в форме удаляет secret из keyring. После этого загрузка работать не будет, пока не введёте ключ заново.

## Стоимость и квоты

Mivra **не делает хеширование** — каждая загрузка создаёт новый объект, даже если вы загрузили тот же скриншот дважды. Это упрощает поведение, но означает, что трафик/storage у провайдера растёт линейно.

При активном использовании скриншотов следите за биллингом. У всех провайдеров есть бесплатный тариф (Yandex — 10 GB, TimeWeb — зависит от тарифа, AWS — 5 GB free tier на 12 месяцев).

## Troubleshooting

### «Тест соединения» падает с 403

- Проверьте Access Key ID и Secret — скорее всего, опечатка.
- Убедитесь, что у ключа есть права на нужный bucket.
- Для **TimeWeb**: точно ли вы взяли **S3** ключи, а не Swift?

### «Тест соединения» падает с 404

- Bucket не найден. Проверьте имя bucket'а (для Yandex это часто длинный ID, не короткое имя).
- Проверьте регион — у разных провайдеров разные коды (`ru-central1` у Yandex, `ru-1` у TimeWeb, `us-east-1` у AWS).

### Загрузка проходит, но ссылка даёт 403 в браузере

- Bucket не настроен на публичное чтение. См. шаг 2 выше.
- Если используете **Public URL prefix** — проверьте, что он соответствует реальному публичному адресу bucket'а у провайдера.

### Несколько секунд «Загрузка...» и потом «Ошибка: network»

- Нет интернета или firewall блокирует HTTPS на S3-эндпоинт.
- Mivra делает 3 попытки с паузами 1с/2с/4с, потом выдаёт ошибку. Если стабильно падает — проверьте подключение или попробуйте другой провайдер.

### Кнопка «S3» снова стала серой после редактирования настроек

Это **by design**: любое изменение поля в форме сбрасывает «verified» статус. Нажмите снова «Тест соединения» → «Сохранить», чтобы кнопка снова стала зелёной.

## Что сейчас не поддерживается

- **Удаление** загруженных файлов из bucket'а через UI — пока только напрямую в панели провайдера.
- **Прогресс-бар** для крупных файлов (>10 MB) — показывается только индикатор «Загрузка...» без процентов.
- **Приватные bucket'ы с presigned URLs** — Mivra работает только с публичными.
- **Миграция** уже-загруженных URL'ов при смене провайдера — нужно перенести руками.
