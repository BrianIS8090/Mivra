Краткая сводка

Документ описывает текущее состояние системы внешних плагинов Mivra, где плагины устанавливаются как локальные пакеты и запускаются через trusted Plugin API v1.

Ключевые выводы

  • API v1 подходит для внутренних и доверенных плагинов, но не является полноценной песочницей.
  • Плагин должен явно запрашивать permissions, регистрировать cleanup и не ломать глобальные стили приложения.
  • Для публичного каталога нужна будущая версия API v2 с iframe sandbox и RPC-коммуникацией.

Что важно проверить дальше

  • Установку плагина как папки и как `.mivraplugin`.
  • Поведение при выключении, удалении и повторной установке.
  • Ошибки сети, неверный API key и слишком большой документ.
В рабочей версии здесь будет отображаться ошибка OpenRouter, если запрос завершится 401, 402, 429 или сетевым таймаутом.
