# MarkItDown Import Plugin Design

Дата: 2026-06-01

## Цель

Добавить в Mivra внешний плагин `markitdown-import`, который конвертирует выбранный пользователем файл в Markdown через Microsoft MarkItDown и вставляет результат в текущий документ.

Ключевое ограничение: ядро Mivra не меняется. Плагин работает только через существующий `MivraPluginApi` и не добавляет Tauri-команды, permissions, capabilities или изменения backend.

## Область

Входит в первый релиз:

- внешний плагин `plugins/markitdown-import/`;
- пакет `.mivraplugin` для установки через менеджер плагинов;
- выбор файла с диска через `<input type="file">` внутри диалога плагина;
- отправка файла в локальный MarkItDown HTTP-сервис;
- предпросмотр полученного Markdown;
- режимы применения результата:
  - вставить в конец текущего документа;
  - заменить текущий документ;
  - вставить с заголовком и разделителем;
  - скопировать Markdown в буфер обмена без изменения документа;
- понятные ошибки для недоступного сервиса, неуспешной конвертации и пустого результата;
- README для запуска локального сервиса.

Не входит в первый релиз:

- изменения `src/`, `src-tauri/` и официального Plugin API;
- запуск Python-процесса из Mivra;
- упаковка Python или MarkItDown в установщик Mivra;
- управление внешними приложениями;
- импорт URL, если файл не выбран с диска;
- точечная вставка в позицию курсора, потому что текущий Plugin API не отдаёт selection.

## Архитектура

Решение состоит из двух частей.

Первая часть - внешний JS-плагин Mivra. Он регистрирует пункт меню `Плагины -> MarkItDown Import`, открывает диалог, читает выбранный файл как `File`, отправляет его в локальный сервис, показывает Markdown и применяет результат через `api.document.setContent(...)`.

Вторая часть - локальный HTTP-сервис из пакета плагина, например `server/markitdown_server.py`. Пользователь запускает его отдельно. Сервис принимает `multipart/form-data`, конвертирует загруженные байты через Python API MarkItDown и возвращает JSON:

```json
{
  "markdown": "# Converted document\n\nText",
  "fileName": "source.docx"
}
```

Сервис слушает только `127.0.0.1`, чтобы не открывать сетевой доступ наружу. Плагин по умолчанию обращается к `http://127.0.0.1:8765/convert`. Endpoint можно переопределить в поле настроек диалога и сохранить в `localStorage`.

## Plugin Manifest

Плагин использует минимальные существующие permissions:

```json
{
  "id": "markitdown-import",
  "name": "MarkItDown Import",
  "version": "1.0.0",
  "description": "Конвертирует выбранный файл в Markdown через локальный MarkItDown-сервис и вставляет результат в документ.",
  "author": "Mivra Team",
  "entry": "index.js",
  "styles": "style.css",
  "permissions": ["document:read", "document:write", "dialog"],
  "apiVersion": 1
}
```

`document:read` нужен для формирования нового содержимого при вставке в конец или с разделителем. `document:write` нужен для применения результата. `dialog` нужен для UI.

## Поток Данных

1. Пользователь открывает пункт `MarkItDown Import`.
2. Плагин показывает диалог с выбором файла, endpoint, режимом применения и областью предпросмотра.
3. Пользователь выбирает файл.
4. Плагин отправляет файл на локальный endpoint методом `POST`.
5. Сервис конвертирует файл через MarkItDown и возвращает Markdown.
6. Плагин сохраняет результат в состоянии диалога и показывает предпросмотр.
7. Пользователь выбирает действие применения.
8. Плагин либо вызывает `api.document.setContent(nextContent)`, либо пишет результат в `navigator.clipboard`.

## Правила Применения

`Заменить документ`:

```text
convertedMarkdown
```

`Вставить в конец`:

```text
currentMarkdown

convertedMarkdown
```

`Вставить с заголовком и разделителем`:

```text
currentMarkdown

---

## Импортировано из source.docx

convertedMarkdown
```

Если текущий документ пустой, режимы вставки не добавляют лишние пустые строки в начало.

## Ошибки

Диалог показывает ошибки без изменения документа:

- сервис недоступен: предложение запустить локальный сервер;
- HTTP-ответ не `2xx`: статус и текст ошибки;
- JSON не содержит непустой `markdown`;
- файл не выбран;
- буфер обмена недоступен: предложение вручную скопировать из предпросмотра.

Плагин не должен автоматически применять результат после конвертации. Применение всегда отдельное действие пользователя.

## Безопасность

Плагин доверенный, как и остальные плагины API v1. При этом локальный сервис ограничивается следующими правилами:

- bind только на `127.0.0.1`;
- принимать только `POST /convert`;
- не принимать произвольные пути к файлам от клиента;
- конвертировать только байты загруженного файла;
- возвращать только Markdown и метаданные без записи результата на диск.

MarkItDown выполняет I/O с правами процесса, поэтому сервис не должен принимать remote URL или path-параметры в первом релизе.

## Тестирование

Тесты первого релиза:

- manifest содержит ожидаемые поля и permissions;
- `activate` регистрирует renderer-диалог и пункт меню;
- диалог рендерит выбор файла, endpoint, режимы применения и кнопки;
- успешная конвертация показывает предпросмотр;
- `Заменить документ` вызывает `api.document.setContent(markdown)`;
- `Вставить в конец` сохраняет текущий документ и добавляет Markdown без лишних пустых строк;
- `Вставить с заголовком и разделителем` добавляет имя файла в заголовок;
- `Скопировать` вызывает `navigator.clipboard.writeText`;
- недоступный сервис показывает ошибку и не меняет документ.

Проверки перед завершением реализации:

- `npm run test`;
- `npx tsc --noEmit`;
- тест локального Python-сервиса на простой текстовый файл;
- `graphify update .` после code-правок.

## Принятые Решения

Основной endpoint первого релиза: `http://127.0.0.1:8765/convert`.

Если CORS в Tauri renderer заблокирует запрос к localhost, это решается настройкой локального сервиса: он возвращает `Access-Control-Allow-Origin: *` и обрабатывает preflight `OPTIONS`.

Если в будущем понадобится полностью автономный импорт без ручного запуска сервиса, это будет отдельная задача и потребует изменений ядра Mivra или отдельного установленного helper-приложения.
